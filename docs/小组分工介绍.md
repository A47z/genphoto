# GenPhoto 小组分工介绍文档

## 项目概述

GenPhoto 是一个基于 **Spring Boot 3 + React** 的 AI 图片生成平台，采用 Maven 多模块架构，包含用户认证、AI 图片生成、图片管理、社交分享、收藏标签、统计分析等功能。

**技术栈：** Spring Boot 3.4.3 / Spring Security / Spring AI / LangChain4J / JPA + MySQL / MinIO / JWT / React + TypeScript + Vite

**模块总览：**

| 模块 | 功能 |
|------|------|
| genphoto-common | 公共工具：统一响应、异常处理、事件定义 |
| genphoto-web | 主启动模块：聚合所有子模块、配置管理 |
| genphoto-user | 用户注册登录、JWT 认证、Spring Security |
| genphoto-image | 图片存储管理、MinIO 对象存储集成 |
| genphoto-generation | AI 图片生成、Spring AI 集成 |
| genphoto-prompt | 提示词模板管理、变量渲染 |
| genphoto-social | 公开分享、评论系统、事件驱动 |
| genphoto-ai | AI 文本服务：灵感生成、提示词优化、自动评价 |
| genphoto-favorite | 收藏夹、标签分类系统 |
| genphoto-stats | 数据统计、报表导出 |
| genphoto-frontend | React 前端应用 |

## 成员 1：项目架构与公共模块

**负责模块：** `genphoto-common` + `genphoto-web`

### 介绍要点

#### 1. 项目整体架构设计

- 采用 **Maven 多模块架构**，将业务按领域拆分为独立模块，各模块职责清晰、低耦合
- 主模块 `genphoto-web` 通过 pom.xml 聚合所有子模块，统一依赖版本管理
- 启动类 `GenPhotoApplication` 使用 `@EnableAsync` 支持异步事件处理

```
genphoto/
├── genphoto-common       ← 公共基础层
├── genphoto-web          ← 聚合启动层
├── genphoto-user         ← 用户认证
├── genphoto-image        ← 图片管理
├── genphoto-generation   ← AI 生成
├── genphoto-prompt       ← 提示词模板
├── genphoto-social       ← 社交分享
├── genphoto-ai           ← AI 文本
├── genphoto-favorite     ← 收藏标签
├── genphoto-stats        ← 数据统计
└── genphoto-frontend     ← 前端应用
```

#### 2. 统一响应封装（genphoto-common）

- **`Result<T>`** — 统一 API 响应格式，包含 `code`、`message`、`data` 三个字段
  - `Result.ok(data)` 返回成功响应
  - `Result.error(message)` 返回错误响应
- **`PageResult<T>`** — 分页响应封装，包含 `items`、`total`、`page`、`size` 等分页元数据
- 所有 Controller 统一返回 `Result<T>` 或 `Result<PageResult<T>>`，前端无需判断不同格式

#### 3. 全局异常处理

- **`BusinessException`** — 自定义业务异常，携带错误消息
- **`GlobalExceptionHandler`** — 使用 `@RestControllerAdvice` 统一拦截异常
  - 捕获 `BusinessException` 返回业务错误
  - 捕获 `MethodArgumentNotValidException` 返回参数校验错误
  - 捕获 `Exception` 兜底处理未知异常
- 避免在每个 Controller 中重复 try-catch，代码更简洁

#### 4. 事件驱动机制

- **`ImagePublishedEvent`** — 当图片被设为公开时发布 Spring 事件
- 利用 Spring 的 `ApplicationEventPublisher` 实现模块间解耦通信
- social 模块发布事件 → ai 模块监听事件 → 自动生成 AI 评价
- 使用 `@Async` 异步处理，不阻塞主流程

#### 5. 全局配置管理（application.yml）

- 数据库连接、JPA 配置（ddl-auto: update）
- MinIO 存储配置（endpoint、access-key、bucket）
- Spring AI 配置（模型、API 地址）
- JWT 密钥与过期时间
- 文件上传大小限制（10MB）

## 成员 2：用户认证与安全模块

**负责模块：** `genphoto-user`

### 介绍要点

#### 1. 用户实体与数据模型

- **User 实体**：`id`、`email`、`passwordHash`、`nickname`、`avatarUrl`、`role`、`createdAt`
- 使用 JPA `@Entity` 映射到 `users` 表
- 密码使用 **BCrypt** 加密存储，不存明文
- `@PrePersist` 自动填充创建时间

#### 2. 用户注册与登录

- **注册流程：** 邮箱去重校验 → BCrypt 加密密码 → 保存用户 → 返回成功
- **登录流程：** 邮箱查询用户 → BCrypt 验证密码 → 生成 JWT Token → 返回 Token
- `UserService` 处理核心业务逻辑
- `UserRepository` 提供 `findByEmail`、`existsByEmail` 等数据访问方法

#### 3. JWT Token 认证机制

- **`JwtTokenProvider`** — JWT 工具类
  - 使用 **HMAC-SHA256** 算法签名
  - Token 有效期 **24 小时**（86400000ms）
  - `generateToken(userId)` → 生成 Token
  - `getUserIdFromToken(token)` → 解析用户 ID
  - `validateToken(token)` → 校验 Token 有效性

#### 4. Spring Security 安全配置

- **`SecurityConfig`** — 核心安全配置类
  - 禁用 CSRF（前后端分离，使用 JWT）
  - 会话策略设为 **STATELESS**（无状态）
  - 公开接口白名单：登录、注册、公共图片、Swagger 文档
  - 其他接口均需认证
- **`JwtAuthenticationFilter`** — JWT 请求过滤器
  - 从 `Authorization: Bearer <token>` 头中提取 Token
  - 验证 Token 并设置 Spring Security 上下文
  - 在 `UsernamePasswordAuthenticationFilter` 之前执行

#### 5. 用户资料管理

- `GET /api/user/profile` — 获取当前用户信息
- `PUT /api/user/profile` — 修改昵称、头像
- `PUT /api/user/password` — 修改密码（需验证旧密码）

#### 6. CORS 跨域配置

- 允许前端开发地址 `http://localhost:5173` 和生产地址 `http://your-server-ip`
- 允许 GET/POST/PUT/DELETE/OPTIONS 方法
- 允许携带凭证（Credentials）

### API 端点

| 方法 | 路径 | 说明 | 是否需认证 |
|------|------|------|-----------|
| POST | /api/user/register | 用户注册 | 否 |
| POST | /api/user/login | 用户登录 | 否 |
| GET | /api/user/profile | 获取个人资料 | 是 |
| PUT | /api/user/profile | 修改个人资料 | 是 |
| PUT | /api/user/password | 修改密码 | 是 |

## 成员 3：图片管理与对象存储模块

**负责模块：** `genphoto-image`

### 介绍要点

#### 1. 图片实体与数据模型

- **Image 实体**：`id`、`userId`、`prompt`、`filePath`、`fileSize`、`width`、`height`、`model`、`style`、`isPublic`、`createdAt`
- `filePath` 存储 MinIO 中的对象路径（格式：`用户ID/UUID.png`）
- 支持记录生成参数（模型、风格、尺寸）便于统计分析

#### 2. MinIO 对象存储集成

- **`MinioConfig`** — MinIO 客户端配置
  - Endpoint: `http://your-minio-ip:9000`
  - Bucket: `genphoto-images`
  - 自动初始化 MinIO Client Bean
- **为什么选择 MinIO：**
  - S3 兼容的开源对象存储
  - 适合存储图片等非结构化数据
  - 与数据库解耦，图片不存数据库

#### 3. 图片上传功能

- **用户上传（`upload`）：** 接收 MultipartFile → 生成唯一文件名 → 上传到 MinIO → 保存元数据到数据库
- **AI 生成保存（`saveFromBytes`）：** 接收 byte 数组 → 写入 MinIO → 保存元数据
- **URL 保存（`saveFromUrl`）：** 支持 HTTP URL 和 Base64 Data URI → 下载/解码 → 存入 MinIO
- 文件路径按用户 ID 分目录，避免冲突

#### 4. 图片下载与缓存

- **`GET /api/images/{id}/file`** — 流式返回图片
  - 从 MinIO 获取对象流 → 设置 Content-Type → 流式写入 Response
  - **ETag 缓存**：基于文件路径生成 ETag，支持 `If-None-Match` 返回 304
  - **Cache-Control**：设置 `max-age=604800`（7天缓存）
  - 配合 Nginx 层缓存，实现两级缓存加速

#### 5. 图片删除

- 同时删除 MinIO 中的文件对象和数据库中的元数据记录
- 校验图片归属（只能删除自己的图片）

### API 端点

| 方法 | 路径 | 说明 | 是否需认证 |
|------|------|------|-----------|
| GET | /api/images | 获取我的图片列表（分页） | 是 |
| GET | /api/images/{id} | 获取图片详情 | 是 |
| GET | /api/images/{id}/file | 获取图片文件（流式） | 否 |
| POST | /api/images/upload | 上传自定义图片 | 是 |
| DELETE | /api/images/{id} | 删除图片 | 是 |

## 成员 4：AI 图片生成模块

**负责模块：** `genphoto-generation` + `genphoto-prompt`

### 介绍要点

#### 1. Spring AI 集成（图片生成）

- 使用 **Spring AI** 框架对接 OpenAI 兼容的图片生成 API
- 当前模型：**Gemini 2.5 Flash Image**（支持切换 Gemini 3.1/3 Pro）
- 通过 `ChatModel` 接口发送包含图片生成指令的 Prompt

#### 2. 图片生成流程

```
用户输入 Prompt
    ↓
GenerationController 接收请求
    ↓
ImageGenerationService 构建系统提示词（包含 style、size 等参数）
    ↓
调用 ChatModel.call() → Spring AI → Gemini API
    ↓
解析响应（Media 附件 / Base64 / URL 三种格式）
    ↓
调用 ImageService 保存到 MinIO
    ↓
返回图片访问 URL 给前端
```

#### 3. 模型与风格支持

- **可用模型：** gemini-2.5-flash-image、gemini-3.1-image、gemini-3-pro-image
- **可用风格：** 自然(natural)、鲜艳(vivid)、动漫(anime)、油画(oil-painting)、水彩(watercolor)、像素(pixel-art)
- 风格通过系统提示词注入，引导 AI 生成特定风格的图片

#### 4. 响应解析策略

- **优先级 1：** 检查 Media 附件（Spring AI 标准方式）
- **优先级 2：** 检查 Base64 编码的图片数据
- **优先级 3：** 检查返回的图片 URL
- 使用 `GenerateResult` Record 封装结果（byte 数组或 URL）

#### 5. 提示词模板系统（genphoto-prompt）

- **模板管理：** 用户可创建、查看、删除提示词模板
- **变量渲染：** 模板中使用 `{{变量名}}` 占位，调用时传入变量值替换
- **分类筛选：** 按 category 字段筛选模板
- **提示词历史：** 自动记录用户使用过的 Prompt，支持查看和删除

#### 6. 请求参数

- `GenerateRequest`：prompt（提示词）、model（模型）、style（风格）、width（宽度）、height（高度）
- `GenerateResponse`：imageUrl（图片地址）、prompt、model、width、height

## 成员 5：社交分享与评论模块

**负责模块：** `genphoto-social`

### 介绍要点

#### 1. 图片公开/私密切换

- `PUT /api/share/{imageId}` — Toggle 切换图片可见性
- 校验图片归属权（只能操作自己的图片）
- 图片设为公开时，发布 `ImagePublishedEvent` 事件
- 事件驱动：AI 模块监听事件后自动生成图片评价

#### 2. 公共图库（广场页面）

- `GET /api/share/public` — 分页获取所有公开图片
- `GET /api/share/public/{imageId}` — 获取单张公开图片详情
- **用户信息填充：** 查出图片后，批量查询对应用户的昵称和头像
  - 收集所有 userId → 批量查 User 表 → Map 映射
  - 避免 N+1 查询问题
- 返回 **`PublicImageResponse` DTO**（包含 Image 所有字段 + nickname + avatarUrl）

#### 3. PublicImageResponse DTO 设计

```java
public class PublicImageResponse {
    // Image 所有字段
    private Long id, userId;
    private String prompt, filePath, model, style;
    private Long fileSize;
    private Integer width, height;
    private Boolean isPublic;
    private LocalDateTime createdAt;
    // 用户信息
    private String nickname;
    private String avatarUrl;
}
```

- 使用 `@Builder` 模式构建
- 提供 `from(Image, nickname, avatarUrl)` 静态工厂方法

#### 4. 评论系统

- **Comment 实体：** `id`、`userId`、`imageId`、`content`、`nickname`、`isAiReview`、`createdAt`
- `POST /api/comments/{imageId}` — 添加评论（自动填充当前用户昵称）
- `GET /api/comments/{imageId}` — 分页获取评论列表
- `DELETE /api/comments/{commentId}` — 删除自己的评论
- `isAiReview` 字段区分用户评论和 AI 自动评价

#### 5. 事件驱动 — AI 自动评价

```
用户将图片设为公开
    ↓
ShareService 发布 ImagePublishedEvent
    ↓
ImagePublishedListener（genphoto-ai 模块）异步监听
    ↓
AiTextService 调用 LLM 生成 100-200 字的中文评价
    ↓
评价以 Comment 形式保存（isAiReview = true）
```

## 成员 6：AI 文本服务与收藏标签模块

**负责模块：** `genphoto-ai` + `genphoto-favorite`

### 介绍要点

#### 1. LangChain4J 集成

- 使用 **LangChain4J** 框架对接大语言模型（LLM）
- 当前模型：**GPT-5.4**，通过自定义 API 端点调用
- `LangChain4jConfig` 配置 `ChatLanguageModel` Bean

#### 2. AI 文本功能

- **灵感生成（`POST /api/ai/prompt/inspire`）：**
  - 调用 LLM 随机生成 50-100 字的创意提示词
  - 帮助用户在没有灵感时快速获取创作方向

- **提示词优化（`POST /api/ai/prompt/optimize`）：**
  - 接收用户的原始 Prompt → LLM 优化为 80-150 字的增强版
  - 增加细节描述、构图建议、风格关键词，提升生成质量

- **AI 图片评价（`generateImageReview`）：**
  - 由事件监听器触发（非 API 调用）
  - 根据图片的 Prompt、模型、风格等信息生成 100-200 字中文评价
  - 评价以评论形式保存到数据库（`isAiReview = true`）

#### 3. 事件监听器

- **`ImagePublishedListener`** — 使用 `@EventListener` + `@Async` 异步监听
- 图片公开时自动触发 AI 评价生成
- 异步执行不影响主流程响应速度

#### 4. 收藏夹系统（genphoto-favorite）

- **Favorite 实体：** `id`、`userId`、`imageId`、`createdAt`
- (userId, imageId) 联合唯一约束，防止重复收藏
- `POST /api/favorites` — 添加收藏
- `DELETE /api/favorites/{imageId}` — 取消收藏
- `GET /api/favorites` — 获取我的收藏列表（分页）
- `existsByUserIdAndImageId` 用于判断是否已收藏

#### 5. 标签分类系统

- **Tag 实体：** `id`、`name`（标签名）
- **ImageTag 实体：** `id`、`imageId`、`tagId`（多对多关联表）
- `POST /api/tags` — 给图片添加标签
- `DELETE /api/tags/{imageId}/{tagName}` — 移除标签
- `GET /api/tags` — 获取所有标签
- `GET /api/tags/image/{imageId}` — 获取图片的标签
- `GET /api/images/by-tag` — 按标签筛选图片

## 成员 7：数据统计模块 + 前端开发

**负责模块：** `genphoto-stats` + `genphoto-frontend`

### 介绍要点

#### 1. 统计分析服务（后端）

- **`StatsService`** 提供多维度统计查询
- **个人统计（`GET /api/stats/my`）：**
  - 总生成图片数
  - 本周生成数
  - 最常用模型
  - 最常用风格
- **每日趋势（`GET /api/stats/my/daily`）：**
  - 按日期范围查询每日生成数量
  - 返回 `[{date, count}]` 数组
- **模型分布（`GET /api/stats/my/models`）：**
  - 统计每个模型的使用次数
  - 返回 `[{model, count}]` 用于柱状图展示

#### 2. 全局概览与导出

- **全局统计（`GET /api/stats/overview`）：** 总用户数、总图片数
- **CSV 导出（`GET /api/stats/export`）：**
  - 导出用户的所有生成记录
  - 字段：ID、Prompt、模型、风格、宽度、高度、文件大小、创建时间
  - 设置 `Content-Disposition: attachment` 触发浏览器下载

#### 3. 前端整体架构

- **技术选型：** React 18 + TypeScript + Vite + Zustand + React Router
- **API 层：** Axios 封装，请求拦截器自动注入 JWT Token，响应拦截器处理 401 跳转登录
- **状态管理：** Zustand 管理用户登录状态和生成页面状态
- **路由设计：**

| 路径 | 页面 | 权限 |
|------|------|------|
| /login | 登录注册页 | 公开 |
| /explore | 广场（公共图库） | 公开 |
| / 或 /generate | AI 生成页 | 需登录 |
| /gallery | 我的图库 | 需登录 |
| /stats | 统计面板 | 需登录 |

#### 4. 前端页面介绍

- **LoginPage** — 登录/注册表单，JWT Token 存 localStorage
- **GeneratePage** — 核心页面：左侧 Prompt 输入区（支持 AI 灵感/优化按钮）、模型/风格/尺寸选择器；右侧图片预览区
- **GalleryPage** — 瀑布流图库，支持公开/删除/查看评论等操作
- **ExplorePage** — 公共广场，展示所有公开图片，显示上传者昵称
- **StatsPage** — Bento Grid 布局统计面板，含模型分布柱状图和 CSV 导出

#### 5. 前端设计特色

- **Glass Morphism 设计风格** — 毛玻璃面板、按钮、输入框
- **LazyImage 懒加载组件** — Intersection Observer + 会话级缓存
- **ImageDetailModal 弹窗** — 图片详情 + 评论 + 上传者信息
- **响应式布局** — CSS Grid + Flexbox 适配不同屏幕
- **Lucide React 图标库** — 统一图标风格
